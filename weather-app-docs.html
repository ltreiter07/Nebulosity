WeatherApp Documentation

WeatherApp is a terminal-based weather application built with Tkinter. It fetches real-time weather data using the Tomorrow.io API and allows users to manage a list of favorite cities.

Setup

  1. Install dependencies: 
    pip install dotenv
  2. Install dependencies: 
    pip install requests
  3. Create a .env file in the project root: 
    API_KEY=your_tomorrow_io_api_key
  4. Run the application: 
    python weather_app_tkinter.py

Features

  • Fetches real-time weather data using Tomorrow.io's API.
  • Maps raw weather codes to human-readable descriptions via weathercode.json.
  • Add cities to a favorites list for quick access.
  • Display weather info for selected favorites.
  • Automatically persists favorite cities to disk.

Key Methods

  • get_weather(city, api_key) – Sends a request to Tomorrow.io and returns parsed weather data.
  • WeatherApp.__init__(self, root) – Sets up the Tkinter UI and loads favorites from disk.
  • WeatherApp.fetch_weather(self) – Gets weather for the city in the input box and updates the UI.
  • WeatherApp.add_to_favorites(self) – Adds the entered city to favorites if not already present.
  • WeatherApp.display_favorite_weather(self, event) – Shows weather for the selected city in the listbox.
  • WeatherApp.get_weather(self, city) – Wrapper around global get_weather() using the API key.
  • WeatherApp.update_favorites_list(self) – Refreshes the displayed favorites list in the UI.
  • WeatherApp.save_favorites(self) – Saves the favorites list to favorites.json.
  • WeatherApp.load_favorites(self) – Loads the favorites list from favorites.json (if it exists).

File Structure

  • weather_app_tkinter.py – Main application file containing logic and UI.
  • .env – Contains the Tomorrow.io API key.
  • weathercode.json – Maps Tomorrow.io weather codes to descriptions.
  • favorites.json – Auto-generated; stores user's favorite cities.
